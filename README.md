# OWASP Juice Shop - Manual Bug Reporting

## Project Overview
This repository contains manual bug reports identified while testing the **OWASP Juice Shop** web application. The objective of this project is to practice identifying, documenting, and reporting software defects using industry-standard QA techniques.
The bugs were reproduced, analyzed, and documented with complete details, including reproduction steps, expected and actual results, severity, priority, and supporting screenshots.


## Application Details
- **Application:** OWASP Juice Shop
- **Application Type:** E-commerce Web Application
- **Testing Type:** Manual Testing
- **Testing Technique:** Black Box Testing
- **Browser:** Google Chrome
- **Operating System:** Windows 11

## Project Objectives
- Perform manual testing on the Juice Shop application.
- Identify reproducible functional and usability defects.
- Document bugs using a professional bug report format.
- Assign appropriate severity and priority levels.
- Improve software testing and defect reporting skills.

## Bug Report Format
Each bug report includes:
- Bug ID
- Module
- Environment
- Description
- Preconditions
- Steps to Reproduce
- Expected Result
- Actual Result
- Severity
- Priority
- Status
